# AdmitSee 研究 & PathPal 导师预约 MVP 建议

**日期：** 2026-04-16
**目的：** 研究 AdmitSee 的导师 Profile 设计，为 PathPal 导师预约 MVP 提供方向

---

## 一、AdmitSee 平台研究

### 公司概况
- 2012-2013年创立，由 Lydia Fayal 和 Stephanie Shyu 在宾大法学院期间创办
- 融资 $1.8M 种子轮，服务超过 25万学生
- 核心使命：通过 peer-to-peer 导师和数据驱动洞察，为大学申请带来透明度
- Trustpilot 评分 4.6/5

### Profile 结构

**学术成绩**
- GPA（含排名百分比）
- 标化成绩（SAT/ACT）
- AP/IB 课程
- 高中类型（公立/私立）

**背景信息**
- 家乡、性别、种族
- 母语及其他语言
- 年级

**申请材料（核心卖点）**
- 完整个人陈述（全文）
- 各学校补充文书
- 按主题标记的文书
- 字数统计可见

**课外活动**
- 体育、社团、志愿者、工作/研究经历、荣誉奖项
- 按活动类型标记

**申请结果（关键信任元素）**
- 申请的所有学校及结果（录取/候补/拒绝）
- 申请类型（ED/RD 等）

**建议板块（按主题组织）**
- 推荐人指导建议
- "Why us" 文书策略
- 特定学校录取策略
- 助学金建议
- 面试经验
- 选校方法论
- 校园生活见闻

### 信任机制

1. **身份验证方式：**
   - 大学学生证
   - 大学毕业证
   - 录取通知书或入学确认收据
   - 姓名与支付账户匹配

2. **质量控制：**
   - 团队手动检查可疑 Profile
   - 成绩低于学校25百分位或文书疑似抄袭时，要求补充 CollegeBoard 文件
   - 抄袭检测

3. **隐私保护：**
   - 真实姓名、学生证号、联系方式完全隐藏
   - 仅显示用户名
   - 照片可选

### 商业模式

**导师收入：**
- 50/50 分成（AdmitSee/导师）
- 用户每次解锁 Profile 付 $1-5
- 顶级 Profile 每学期赚超 $500
- 导师辅导费：$5/月/学生
- 高活跃导师获得佣金加成
- $10 注册奖金 + $2/推荐人（上限 $100）

**用户定价：**
- 高三（Class of 2027）：20次解锁 + 消息 + 高级搜索 = $150/年
- 高二（Class of 2028）：35次解锁 = $245/年
- 高一（Class of 2029）：45次解锁 = $300/年
- 初三（Class of 2030）：55次解锁 = $350/年
- 顾问套餐：$600/年（100次解锁）

### 互动流程
1. 免费浏览基础 Profile
2. 付费解锁完整 Profile（$1-5/次）
3. 解锁后可发消息（1对1）
4. 发展持续导师关系

### 用户评价摘录
> "AdmitSee was my college app bible. Real advice and essays made the admissions process much more transparent and less daunting."

> "This site has been infinitely more valuable to me for college applications than any book, any friend, any discussion board, or my high school counselor."

## 二、PathPal vs AdmitSee 差异化定位

| 维度 | AdmitSee | PathPal |
|------|----------|---------|
| 核心定位 | 数据驱动的申请透明化 | 情感陪跑的真人连接 |
| Profile 核心 | 数据+文书+成绩 | **故事+情感+人格** |
| 信任机制 | .edu验证+成绩验证 | .edu验证+**视频自我介绍** |
| 吸引点 | "看看成功者的数据" | **"认识一个真实的学长学姐"** |
| 互动方式 | 解锁Profile → 消息 | **直接预约视频通话** |
| 付费模式 | 订阅制解锁 | **按次预约/陪跑包** |
| 情感维度 | 几乎没有 | **核心差异化** |

**关键结论：AdmitSee 做的是"信息产品"，PathPal 做的是"人的服务"。完全不同的赛道。**

## 三、从 AdmitSee 偷师的好点子

1. **验证徽章** — .edu邮箱 + offer证明，显示在 Profile 上
2. **申请结果展示** — 展示申到哪些学校、被拒哪些（增加真实感）
3. **按话题组织** — "选校策略"、"文书心得"、"面试经验"等标签
4. **隐私保护** — 默认只显示昵称，真名可选
5. **分成激励** — 收入分成让导师有动力维护 Profile 质量
6. **活跃度加权** — 响应快的导师排名更高

## 四、PathPal 导师 Profile 设计建议

### Profile 信息结构

**核心展示区（第一眼看到的）**
- 头像/照片（真人照片，非默认头像）
- 昵称 + 验证徽章
- 学校 + 专业 + 年级
- 一句话标语（如："从上海国际学校到UCLA，我想帮你少走弯路"）
- 30秒视频自我介绍（核心差异化）

**我的故事（情感核心）**
- 团队采访后帮写的个人叙述（300-500字）
- 重点：申请季的真实经历、挫折、成长
- 示例："高三那年我跟我妈几乎天天吵架，SAT考了三次，ED被拒后哭了一整晚..."

**我的申请经历**
- 申请的学校列表 + 结果（录取/候补/拒绝）
- 标化成绩（可选展示）
- 课外活动亮点

**我能帮你什么**
- 明确的服务标签（选校建议、文书反馈、面试模拟、情感支持、校园生活分享）
- 按服务类型列出

**评价和反馈**
- 星级评分 + 评价数
- 真实用户评价（文字）

**预约和定价**
- 服务选择（30分钟/60分钟）
- 价格显示
- 可用时间日历
- 一键预约按钮

## 五、MVP 技术实施建议

### 现有可复用资源

PathPal 已有的代码基础：

| 组件 | 文件路径 | 可复用程度 |
|------|----------|-----------|
| 顾问数据模型 | `src/data/consultants.ts` | 需扩展字段 |
| 顾问 Profile 页 | `src/app/consultant/[id]/page.tsx` | 改造UI风格 |
| 顾问卡片组件 | `src/components/ConsultantCard.tsx` | 改造UI风格 |
| 顾问注册表单 | `src/app/become-consultant/page.tsx` | 可复用表单模式 |
| 预约弹窗流程 | `src/app/consultant/[id]/page.tsx` (lines 265-367) | 加入支付 |
| 用户认证 | `src/context/UserContext.tsx` | 直接复用 |
| Supabase 集成 | `src/lib/supabase/client.ts` | 直接复用 |
| 设计系统 | `src/app/globals.css` (Ivy League蓝+金) | 直接复用 |
| 日期处理 | `date-fns` 已安装 | 直接复用 |

### 数据模型扩展

现有 Consultant 接口需要新增字段：

```typescript
// 新增字段
story: string;              // 个人故事（团队帮写）
video_url: string;          // 30秒视频自我介绍
application_results: {      // 申请结果
  school: string;
  result: 'accepted' | 'waitlisted' | 'denied';
  type: 'ED' | 'EA' | 'RD';
}[];
verified: boolean;          // 是否已验证
verification_type: string;  // 验证方式（.edu / offer证明）
service_tags: string[];     // 服务标签
tagline: string;            // 一句话标语
```

### 需要新建的

**1. Stripe 支付集成**
- 新建 `src/app/api/checkout/route.ts` — Stripe Checkout Session
- 新建 `src/app/api/webhooks/stripe/route.ts` — 支付成功回调
- 安装 `stripe` 和 `@stripe/stripe-js`

**2. Bookings 数据表**
- Supabase 新建 `bookings` 表
- 字段：id, student_id, mentor_id, service_type, date, time, price, status, payment_id, created_at

**3. 通知系统**
- 支付成功后发邮件给双方
- 可用 Resend 或 Supabase Edge Function
- 包含视频通话链接

**4. 导师 Profile 页面改造**
- 重写 `src/app/consultant/[id]/page.tsx`
- 从"顾问简历风"改为"学长学姐故事风"
- 加入视频播放器、个人故事、申请结果展示

### 可以先不做的

- 论坛、AI聊天、大学数据库（与导师预约MVP无关）
- 导师后台/收入看板（早期手动结算）
- 在线评价提交系统（先微信收集）
- 日历同步（iCal/Google Calendar）
- 取消/改期功能（早期人工处理）

### MVP 优先级排序

| 优先级 | 任务 | 预估工作量 |
|--------|------|-----------|
| P0 | 重新设计导师 Profile 页面 | 中 |
| P0 | 接入 Stripe 支付 | 中 |
| P0 | 创建 bookings 表 + 预约持久化 | 小 |
| P1 | 扩展导师数据模型 | 小 |
| P1 | 替换 mock 数据为真实导师 | 小（依赖采访） |
| P1 | 支付成功邮件通知 | 小 |
| P2 | 导师注册表单改造 | 中 |
| P2 | 视频上传/展示功能 | 中 |
| P3 | 评价系统上线 | 中 |
